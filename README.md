# Unity 프레임워크

Unity로 게임을 제작하기 위한 기본 프레임워크입니다. 이 프로젝트는 아키텍처와 필수 시스템을 제공하여 게임 개발을 가속화합니다.

## Unity 버전

이 프로젝트는 **Unity 6** 버전으로 개발 및 테스트되었습니다.

## 핵심 기능

이 프레임워크는 재사용 가능한 여러 시스템을 기반으로 구축되었습니다.

*   **매니저 기반 아키텍처**: 중앙 `Managers` 싱글톤(`@Managers` 게임 오브젝트)을 통해 모든 핵심 시스템에 쉽게 접근할 수 있으며, 코드의 명확성과 관심사 분리를 보장합니다.
*   **비동기 씬 및 리소스 관리**: Unity의 어드레서블 에셋 시스템(Addressable Asset System)을 활용하여 씬과 에셋을 효율적으로 비동기 로딩합니다. 이를 통해 로딩 시간을 단축하고 메모리 사용량을 줄입니다. 이펙트나 발사체와 같은 게임 오브젝트를 재사용하기 위한 풀링 시스템(`PoolManager`)이 포함되어 있습니다.
*   **데이터 기반 설계**: 캐릭터 스탯과 같은 게임 데이터를 외부 JSON 파일(`Assets/03_BundleData/Data`)로 관리합니다. `Data.Contents.cs`에서 새로운 데이터 구조를 쉽게 정의할 수 있으며, 런타임에 딕셔너리로 로드되어 빠른 접근을 지원합니다.
*   **구조적인 UI 시스템**: `UIManager`가 Panel, Popup, Sub-Item을 관리하는 유연한 UI 시스템을 제공합니다. `UI_Base`, `UI_Popup`과 같은 기본 클래스를 통해 새로운 UI 요소를 쉽게 만들 수 있습니다.
*   **상태 머신 플레이어 컨트롤러**: 상태 머신(`Idle`, `Moving`, `Skill`)과 Unity의 NavMeshAgent를 사용하는 견고한 캐릭터 컨트롤러(`PlayerController`)를 통해 클릭 이동 네비게이션을 구현합니다.
*   **이벤트 기반 입력**: 중앙 `InputManager`가 원시 입력을 `MouseEvent`와 같은 게임별 이벤트로 변환하여 입력 로직과 게임 플레이 코드를 분리합니다.
*   **확장 가능한 게임 로직**: `GameScene`에서 플레이어와 몬스터를 스폰하는 기본 구현을 포함하여 핵심 게임 플레이 루프를 보여줍니다.

## 프로젝트 구조

프로젝트의 스크립트는 `Assets/02_Scripts/` 아래의 다음 디렉토리로 구성됩니다.

*   **`Contents`**: `PlayerStat`과 같이 게임 고유의 로직 및 데이터 컴포넌트를 포함합니다.
*   **`Controller`**: 게임 오브젝트의 동작을 제어하는 `MonoBehaviour` 스크립트(`PlayerController`, `MonsterController`, `CameraController` 등)를 포함합니다.
*   **`Data`**: 외부 JSON 데이터 파일에 매핑되는 C# 데이터 구조를 정의합니다.
*   **`Editor`**: WebGL 빌드 헬퍼와 같이 에디터의 기능을 확장하는 커스텀 스크립트를 포함합니다.
*   **`Managers`**: 프레임워크의 핵심입니다. 각 매니저는 특정 시스템을 담당하는 싱글톤입니다.
*   **`Scenes`**: 특정 씬의 흐름과 로직을 제어하는 스크립트(`GameScene`, `LoginScene` 등)입니다. `BaseScene`을 상속받습니다.
*   **`UI`**: `Panel`, `Popup`, `SubItem` 하위 디렉토리로 구성된 모든 UI 요소 관련 스크립트를 포함합니다.
*   **`Utils`**: 상수 및 열거형을 위한 `Define.cs`, C# 확장 메서드 등 유틸리티 스크립트 및 헬퍼 클래스를 포함합니다.


## 주요 스크립트 및 개념

*   **`Managers.cs`**: 프레임워크의 심장입니다. 다른 모든 매니저를 생성하고 접근을 제공합니다. 모든 씬에서 유지됩니다.
*   **`BaseScene.cs`**: 모든 씬 스크립트의 기본 클래스입니다. 씬이 로드될 때 `InitAsync()` 메서드가 호출되도록 보장하며, 씬별 로직을 시작하기에 이상적인 장소입니다.
*   **`PlayerController.cs`**: 상태 기반 컨트롤러 패턴의 명확한 예시입니다. 마우스 입력을 처리하여 이동하고 공격합니다.
*   **`Data.Contents.cs`**: JSON에서 로드할 수 있는 게임 데이터를 담기 위해 직렬화 가능한 클래스를 정의하는 방법을 보여줍니다.
